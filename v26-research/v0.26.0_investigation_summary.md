# v0.26.0-nightly Investigation: Main New Features

## 1. skill-creator (PR #16394)
**What it is:** A built-in skill that automates the creation, validation, and packaging of custom skills.

**How it works:**
- `skill-creator` Skill: Provides procedural guidance for designing and implementing effective skills.
- `init_skill.cjs`: Automatic scaffolding of new skills with standard templates.
- `validate_skill.cjs`: Validates YAML frontmatter and unresolved TODOs.
- `package_skill.cjs`: Packages skills into `.skill` files without redundant nesting.

**Benefits:**
- Automates the entire skill lifecycle.
- Standardizes skill creation.
- Agents can install skills automatically on behalf of the user.
- Improves the quality and consistency of new skills.

## 2. Experimental Plan Flag (PR #16650)
**What it is:** An experimental flag `experimental.plan` that controls all planning-related features.

**How it works:**
- Configured in `settings.json` under `experimental.plan`.
- Controls the 'Plan' approval mode and specialized planning tools.
- Allows enabling/disabling planning functionalities as needed.

**Benefits:**
- Paves the way for future planning features.
- Allows controlled experimentation.
- Users can enable/disable according to their workflow.

## 3. Security Consent Prompts for Skills (PR #16549)
**What it is:** Interactive security consent prompts for skill installation.

**How it works:**
- When you run `gemini skills install`, it shows a consent prompt.
- Shows which skill is being installed, where it comes from, and where it will be installed.
- Includes critical security warnings.
- `--consent` flag for non-interactive environments.

**Benefits:**
- Improves security: users are aware of risks.
- Improved visual clarity.
- Support for automation with `--consent`.
- Consent logging in debug logs.

## 4. Agents in settings.json (PR #16433)
**What it is:** Support for configuring agents directly in `settings.json`.

**How it works:**
```json
{
  "agents": {
    "codebaseInvestigator": {
      "modelConfig": {
        "temperature": 1
      },
      "runConfig": {
        "maxTurns": 100
      }
    }
  }
}
```

**Benefits:**
- Granular customization of agent behavior.
- Enable/disable individual agents.
- Override of `runConfig` and `modelConfig` per agent.
- Centralized configuration control.
